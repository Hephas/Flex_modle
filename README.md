# AEM 電解槽熱管理與廢熱回收模型 - 程式設計說明與操作手冊

## 1. 系統概述與軟體架構
本軟體旨在解決 AEM 電解系統在不同功率負載下，熱交換幫浦因「最低流量限制 (30 LPM)」造成的系統過度冷卻與熱力學不匹配問題。
系統採用**「灰盒模型 (Gray-box Model)」**，結合理論熱力學方程式與實際設備規格書（AEM 階梯式能耗），並具備**「人機協作 (Human-in-the-loop)」**的雙引擎架構：
* **預測引擎 (前饋控制)：** 讀取即時感測器狀態，利用設定的換熱效能預先計算出下一秒的最佳幫浦流量，避免溫度失控。
* **診斷引擎 (事後校準)：** 接收換熱器出口的真實溫度，反向推算實際的熱交換效能，供工程師手動校準預測模型，實現自適應修正。

## 2. 核心熱力學演算法
軟體內部的運算核心基於以下三組物理化學方程式：

### A. 動態廢熱預測 (AEM 規格書查表)
系統根據輸入功率 $EL\_PWR$ 判斷負載區間，選用對應的系統能耗（4.8, 5.0, 5.2 kWh/Nm³），並以水裂解理論最低能耗 3.54 kWh/Nm³ 為基準計算發熱比例：
$$Q_{load} = EL\_PWR \times \left( 1 - \frac{3.54}{Energy\_Req} \right)$$

### B. 前饋最佳流量計算
基於能量守恆，計算能完美帶走 $Q_{load}$ 的冷水流量：
$$HR\_PF_{ideal} = \frac{60 \cdot Q_{load}}{4.18 \cdot \epsilon_{setting} \cdot (EL\_HT - HR\_LT)}$$

### C. 實測熱交換效能 ($\epsilon$) 診斷
依據 $\epsilon-NTU$ 理論，計算實際換熱量與最大理論換熱潛力的比值：
$$Q_{cold\_gained} = \left( \frac{Current\_HR\_PF}{60} \right) \cdot 4.18 \cdot (HR\_HT - HR\_LT)$$
$$\epsilon_{actual} = \frac{Q_{cold\_gained}}{C_{min} \cdot (EL\_HT - HR\_LT)}$$

## 3. 軟體介面與參數定義 (Data Dictionary)

本軟體介面分為三個主要區塊與一個側邊欄設定區。以下為各參數的定義與邊界條件：

### 側邊欄：系統常數與模型設定 (靜態邊界)
| 變數名稱 | 說明 | 單位 / 預設值 | 系統作用 |
| :--- | :--- | :--- | :--- |
| `T_target` | 目標控制溫度 | 55.0 °C | 系統期望維持的最佳電解液溫度 |
| `T_min` | 強制關閉保護溫度 | 53.0 °C | 低於此值強制輸出 0 LPM，保護電解效能 |
| `PF_min` | 幫浦最低流量 | 30.0 LPM | 硬體限制，避免馬達損壞 |
| `PF_max` | 幫浦最大流量 | 100.0 LPM | 硬體限制的最大冷卻能力 |
| `epsilon_setting`| **設定換熱器效能 ($\epsilon$)** | 0.85 (10%-100%) | **前饋運算的核心常數，需依據診斷結果微調** |
| `eff_low/mid/high`| AEM 階梯能耗 | 5.2 / 5.0 / 4.8 | 依據規格書設定，定義不同負載的產熱率 |

### 區塊一：當前感測器狀態 (動態輸入)
| 變數名稱 | 說明 | 單位 | 系統作用 |
| :--- | :--- | :--- | :--- |
| `EL_PWR` | 電解槽輸入功率 | kW | 決定動態產熱量 $Q_{load}$ |
| `EL_HT` | 電解液進入換熱前高溫 | °C | 熱源溫度，計算溫差的基準 |
| `HR_LT` | 熱回收冷水溫度 | °C | 冷源溫度，決定冷卻潛力 |
| `EL_PF` | 當前電解液流量 | LPM | 用於事後計算熱水側熱容率 |
| `current_HR_PF`| 當前換熱幫浦流量 | LPM | 用於事後計算冷水側熱容率 |

### 區塊二：預測控制輸出 (運算結果)
| 變數名稱 | 說明 | 狀態 / 意義 |
| :--- | :--- | :--- |
| `target_flow` | **建議下一步換熱流量** | 系統給出的最佳 HR-PF 指令 (受限於 30-100 LPM) |
| `q_load` | 預測系統廢熱 | 當前功率下的理論發熱量 (kW) |
| `msg` | 系統診斷訊息 | 包含：正常控溫、冷卻過剩警告、散熱不足警告 |

### 區塊三：事後效能診斷 (驗證輸入與結果)
| 變數名稱 | 說明 | 意義與後續行動 |
| :--- | :--- | :--- |
| `EL_LT` (輸入) | 換熱後回電解槽低溫 | 觀察電解液實際降了多少度 |
| `HR_HT` (輸入) | 換熱後熱回收高溫 | 觀察冷水實際升了多少度 (熱回收品質) |
| `Q_hot_removed`| 電解側實際被帶走熱量 | 若大於冷側獲得熱量，代表環境散熱嚴重 |
| `Q_cold_gained`| 熱回收實際獲得熱量 | 實際進入熱水桶的能量 |
| `actual_epsilon`| **實測換熱器效能 ($\epsilon$)**| **將此數值回饋至側邊欄 `epsilon_setting` 進行校準** |

## 4. 標準操作流程 (SOP)

操作人員在進行機台帶載測試時，請遵循以下四個步驟使用本軟體：

1. **初始設定確認：** 開機前，確認左側邊欄的「系統滿載功率」、「幫浦流量極限」與「AEM 能耗基準」是否與當前設備硬體相符。初次運行請將 `設定換熱器效能 (ε)` 設為 **0.85**。
2. **輸入當下狀態 (區塊一)：** 系統開始產氫並達到熱平衡後，將儀表板上的 $EL\_PWR$、$EL\_HT$、$HR\_LT$、$EL\_PF$、$current\_HR\_PF$ 數值輸入至網頁中。
3. **執行流量控制 (區塊二)：** 讀取模型算出的「建議下一步換熱流量」。
   * 若顯示「✅ 正常控溫中」，請將幫浦手動調整至該數值。
   * 若顯示「⚠️ 冷卻過剩警告」，代表即使開最低流量 (30 LPM) 也會造成系統降溫，請採取手動間歇啟停（開幾秒後關閉）或開啟旁路閥。
4. **事後診斷與模型校準 (區塊三)：** 流量調整完畢且溫度穩定後，讀取換熱器出口的 $EL\_LT$ 與 $HR\_HT$ 溫度並輸入系統。查看算出的**「實測換熱器效能 (ε)」**。將此實測數值輸入回左側邊欄的 `設定換熱器效能 (ε)` 滑桿中，完成本次模型校準。

## 5. 工程風險與系統限制宣告

* **流量不匹配死區：** 本模型已透過數學證明，在低載或冷水溫度過低的情境下，硬體最低 30 LPM 的冷卻能力遠大於 AEM 發熱量。這屬於硬體層級的物理限制，軟體無法憑空消除降溫現象。強烈建議未來在管路上加裝機械式旁路分流閥 (Bypass Valve) 或改裝低頻變頻器。
* **感測器依賴性：** 區塊三的效能診斷高度依賴溫度計的精準度。若溫差 $(HR\_HT - HR\_LT)$ 讀值誤差過大，將導致推算出的效能 $\epsilon$ 產生劇烈跳動。請定期校正 PT100 或熱電偶。
